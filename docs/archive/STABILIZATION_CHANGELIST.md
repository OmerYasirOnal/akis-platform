# STABILIZATION CHANGELIST

**Branch:** `feat/scribe-actor-context-and-install-cta`  
**Date:** 2025-10-27  
**Objective:** Fix "User not found" error in Scribe + Add App-Bot Actor + Correct "Install to more repos" CTA with permissions awareness

---

## Summary

This changelist addresses two critical issues:
1. **Scribe "User not found" Error:** Introduced ActorContext system with app_bot fallback to allow Scribe to run headlessly under GitHub App (no OAuth user required)
2. **Install CTA & Permissions:** Fixed "Install to more repos" button to open correct GitHub Installation manage page and surface permissions/repo-coverage gaps with actionable CTAs

## Files Changed

### 1. **NEW: `src/lib/auth/actor.ts`** (New File - 196 lines)
**Purpose:** ActorContext system for authentication mode resolution

**Changes:**
- Added `ActorMode` type: `"oauth_user" | "app_bot" | "service_account"`
- Added `Actor` interface with mode, userId, githubLogin, installationId
- Added `resolveActorOrFallback()` function with intelligent fallback:
  - Priority 1: OAuth user (if available)
  - Priority 2: GitHub App installation (if fallback enabled)
  - Priority 3: Environment variables
- Added `isAppBotFallbackEnabled()` check (feature flag: `SCRIBE_ALLOW_APP_BOT_FALLBACK`, default: true)
- Added `getCommitAuthor()` for attribution based on actor mode
- Added `getActorBanner()` for UI banners
- Added `createOAuthRequiredError()` for actionable errors

**Evidence:**

## Security Considerations

1. **Least Privilege:** App runs with minimum required permissions (see diagnostics endpoint)
2. **Server-Only:** All auth logic remains server-only (no token exposure to client)
3. **Actionable Errors:** Users see "Connect GitHub" or "Install App" CTAs, not raw errors
4. **Audit Trail:** All logs include actor mode and correlation IDs
