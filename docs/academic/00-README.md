## Why In-Repo?

These documents are intentionally kept in the repository to:
- Maintain project lineage and historical context
- Preserve the original scope and requirements for reference
- Provide architectural rationale for future maintainers
- Ensure graduation documentation is version-controlled alongside code

## Excluded Files

For security reasons, the following are **NOT** tracked in git:
- Sensitive credentials (*.pem, *.key, .env files)
- Private keys or certificates
- Personal identification documents
- Any files containing API tokens or passwords

These files are gitignored at the repository root level.

## Notes

- Documents are kept in their original Turkish language as submitted
- PDF versions are provided for read-only reference
- DOCX versions are source documents (may contain tracked changes/comments)
- Markdown versions (where available) are for easy browsing and diff tracking
