## PHASE 1: PhaseProgressBanner Component (Subtask 1/6)

### TASK (WHAT)
Create a `PhaseProgressBanner` component that displays the current agent execution phase as an animated full-width banner with progress bar and estimated time.

### LOCATION (WHERE)
- `frontend/src/components/agents/PhaseProgressBanner.tsx` (NEW)
- `frontend/src/components/agents/__tests__/PhaseProgressBanner.test.tsx` (NEW)
- `frontend/src/i18n/locales/en.json` (ADD keys)
- `frontend/src/i18n/locales/tr.json` (ADD keys)

### RULES (CONSTRAINTS)

**Props interface:**
```typescript
interface PhaseProgressBannerProps {
  currentStage: string | null;       // from useJobStream.currentStage
  stageMessage: string | null;       // from useJobStream.stageMessage
  isRunning: boolean;                // job is in active execution
  isCompleted: boolean;              // job reached terminal state
  isFailed: boolean;                 // job failed
  qualityScore?: number | null;      // 0-100, shown on completion
  startedAt?: number;                // timestamp for duration calculation
}
```

**Agent phases (in order):**
```
init → planning → executing → reflecting → validating → publishing → completed
```

**Display behavior:**
1. When `isRunning=false` and not completed: Show "Ready to run" state (muted)
2. When `isRunning=true`: Show animated banner with:
   - Phase icon (use emoji or Lucide icons): 🧠 init, 📋 planning, ⚡ executing, 🔍 reflecting, ✅ validating, 📤 publishing
   - Phase name in bold (i18n)
   - One-line stage message (from `stageMessage`, or i18n default)
   - Progress bar: calculate percentage from `currentPhaseIndex / totalPhases`
   - Elapsed time: `Math.floor((Date.now() - startedAt) / 1000)` seconds, update every second
   - Animated pulse effect on the active phase icon
3. When `isCompleted=true`: Show success banner with quality score badge
4. When `isFailed=true`: Show error banner with red background

**Visual design:**
- Full-width, 64px height
- Background: `bg-ak-surface` with left border color indicating status (green=running, blue=completed, red=failed)
- Progress bar: thin (4px) at bottom, `bg-ak-primary` fill with CSS transition
- Quality score: circular badge (32px) on the right side, color-coded (green ≥70, yellow ≥40, red <40)

## PHASE 2: InnerMonologue Component (Subtask 2/6)

### TASK (WHAT)
Create an `InnerMonologue` component that displays the agent's reasoning and actions as streaming typewriter text, showing what the agent is "thinking" in real-time.

### LOCATION (WHERE)
- `frontend/src/components/agents/InnerMonologue.tsx` (NEW)
- `frontend/src/components/agents/__tests__/InnerMonologue.test.tsx` (NEW)
- `frontend/src/hooks/useTypewriter.ts` (NEW — reusable typewriter hook)

### RULES (CONSTRAINTS)

**Props interface:**
```typescript
interface InnerMonologueProps {
  events: StreamEvent[];      // from useJobStream.events
  traceEvents: TraceStreamEvent[];  // from useJobStream.traceEvents
  stageEvents: StageStreamEvent[];  // filtered stage events
  isRunning: boolean;
  maxDisplayedMessages?: number;  // default: 20, for performance
}
```

**useTypewriter hook:**
```typescript
function useTypewriter(text: string, speed?: number): {
  displayedText: string;
  isTyping: boolean;
}
```
- Default speed: 30ms per character (adjustable)
- Use `requestAnimationFrame` for smooth rendering — NOT `setInterval`
- When new text arrives before current animation finishes, immediately show remaining old text and start animating new text
- Respect `prefers-reduced-motion` media query — if enabled, show text instantly

**Message generation logic:**
Convert trace events into human-readable "inner monologue" messages:

| Event Type | Generated Message |
|---|---|
| `stage` (planning, started) | "🧠 Planning how to approach this task..." |
| `trace` (doc_read) | "📄 Reading `{title}`..." |
| `trace` (file_created) | "✏️ Generating `{title}`..." |
| `trace` (reasoning) | "💭 {reasoningSummary}" |
| `trace` (decision) | "🎯 {title}: {detail}" |
| `tool` (ok=true) | "🔧 {did}" (from tool's `did` field) |
| `tool` (ok=false) | "⚠️ Tool call failed: {errorSummary}" |
| `aiCall` (ok=true) | "🤖 AI responded ({model}, {tokens} tokens, {durationMs}ms)" |
| `artifact` (file) | "📦 Produced: `{label}`" |
| `artifact` (doc_read) | "📖 Read: `{label}`" |
| `stage` (completed) | "✅ Task completed successfully!" |
| `stage` (failed) | "❌ Execution failed: {message}" |

**Display behavior:**
1. Messages appear in a scrollable container (max-height: 300px)
2. Latest message animates with typewriter effect
3. Previous messages are fully displayed (no animation)
4. Auto-scroll to bottom when new messages arrive
5. Each message has a subtle timestamp on the right (HH:MM:SS)
6. Phase transition messages have a divider line above them

**Visual design:**
- Container: `bg-ak-surface` with `rounded-lg`, padding 16px
- Text: `font-mono text-sm text-ak-text-secondary`
- Active message: `text-ak-text` (brighter)
- Typewriter cursor: blinking `|` character (CSS animation)
- Phase dividers: thin `border-ak-border` line with phase label centered

**Accessibility:**
- Container has `role="log"` and `aria-live="polite"`
- Each message is a `<div role="status">`
- Typewriter animation respects `prefers-reduced-motion`

**Tests:** At least 3 tests:
1. Renders messages from trace events
2. Typewriter effect animates text progressively
3. Auto-scrolls to bottom on new messages

## PHASE 5: LiveAgentCanvas Integration (Subtask 5/6)

### TASK (WHAT)
Create the `LiveAgentCanvas` — the unified container that combines PhaseProgressBanner, InnerMonologue, PhaseActivityCards, and ExpandingFileTree into a single cohesive real-time execution view. Then integrate it into all three agent console pages.

### LOCATION (WHERE)
- `frontend/src/components/agents/LiveAgentCanvas.tsx` (NEW)
- `frontend/src/components/agents/__tests__/LiveAgentCanvas.test.tsx` (NEW)
- `frontend/src/pages/dashboard/agents/DashboardAgentScribePage.tsx` (MODIFY — replace Logs tab content)
- `frontend/src/pages/dashboard/agents/trace/index.tsx` (MODIFY — replace Logs tab content)
- `frontend/src/pages/dashboard/agents/proto/index.tsx` (MODIFY — replace Logs tab content)

### RULES (CONSTRAINTS)

**Props interface:**
```typescript
interface LiveAgentCanvasProps {
  jobId: string | null;
  isRunning: boolean;
  isCompleted: boolean;
  isFailed: boolean;
  qualityScore?: number | null;
  startedAt?: number;
  // View mode
  defaultView?: 'stream' | 'timeline' | 'quality';
}
```

**Layout:**
```
┌──────────────────────────────────────────┐
│ PhaseProgressBanner (always visible)     │
├──────────────────────────────────────────┤
│ [Stream] [Timeline] [Quality]  ← tabs   │
├──────────────────────────────────────────┤
│                                          │
│ Stream View:                             │
│   InnerMonologue (top 60%)               │
│   ─────────────────────                  │
│   PhaseActivityCards (bottom 40%)        │
│                                          │
│ Timeline View:                           │
│   Enhanced StepTimeline (full height)    │
│                                          │
│ Quality View:                            │
│   QualityDashboard (post-completion)     │
│   ExpandingFileTree                      │
│                                          │
└──────────────────────────────────────────┘
```

**View switching:**
- During execution: default to "Stream" view
- On completion: auto-switch to "Quality" view
- User can switch views at any time
- View preference persisted in localStorage

**Integration pattern:**
In each agent console page, replace the current `<Logs>` tab content:

```tsx
// BEFORE (current):
<div className="logs-tab">
  {traceEvents.map(event => <LogEntry ... />)}
</div>

// AFTER (enhanced):
<LiveAgentCanvas
  jobId={currentJobId}
  isRunning={jobState === 'running'}
  isCompleted={jobState === 'completed'}
  isFailed={jobState === 'failed'}
  qualityScore={job?.qualityScore}
  startedAt={jobStartTime}
/>
```

**DO NOT remove existing tabs** (Preview, Diff, Results, Artifacts). Only replace the Logs tab content. Other tabs remain unchanged.

**Tests:** At least 3 tests:
1. Renders all sub-components when running
2. Switches to Quality view on completion
3. View tabs work correctly
