# Contributing Guide

Thanks for contributing! Please:
1. Open an issue describing the change.
2. Create a feature branch from `main`.
3. If you change behavior or public API, update docs accordingly.
4. Keep PRs small and focused. Reference issues in PR body.

## Documentation Rules
- Update `README.md` and `docs/GETTING_STARTED.md` for any developer-facing change.
- Add/extend `docs/ARCHITECTURE.md` for structural changes.
- If you introduce endpoints, update `docs/API.md`.

## Commit & PR
- Conventional-like titles preferred: `docs:`, `feat:`, `fix:`, `chore:`.
- Include screenshots/logs if applicable.
